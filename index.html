MuleSoft Gemini Agentic PAPI

Process API — Orchestration Layer

A MuleSoft Process API that orchestrates business logic for the Gemini Agentic commerce platform. Routes requests from the Experience API to SAP System APIs with transformation, validation, and error handling.

Architecture

This PAPI sits in the middle of the API-led connectivity stack — receiving requests from the EAPI and routing to the appropriate System APIs.

Features

🔀 Smart Routing

Route incoming requests to Cart SAPI or Checkout SAPI based on operation type and business rules.

🔄 Transformation

Translate EAPI payloads into SAPI-compatible formats using DataWeave transformations.

⚡ Business Logic

Apply centralised business rules — pricing, validation, eligibility checks — before routing.

🛡️ Error Handling

Centralised error management across all downstream SAPIs with consistent error responses.

Getting Started

Step 1 — Clone the repository:

git clone https://github.com/viplgoswami1/mulesoft-gemini-agentic-papi.git
cd mulesoft-gemini-agentic-papi

Step 2 — Import into Anypoint Studio:

File → Import → Anypoint Studio Project from File System → Select cloned folder → Finish

Step 3 — Configure in src/main/resources/config.yaml:

cart:
  sapi:
    url: "http://localhost:8083"
checkout:
  sapi:
    url: "http://localhost:8084"
http:
  listener:
    port: 8082

API Endpoints

POST /cart

Create a cart session. Validates, transforms and routes to Cart SAPI.

PUT /cart/{cartId}

Update an existing cart. Routes update to Cart SAPI with full line item replacement.

POST /checkout

Initiate checkout. Routes to Checkout SAPI for SAP order creation.
